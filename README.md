The INO code is uploaded via the Arduino IDE onto the esp32 using the ESP32 board setting and also the Adafruit libraries required for the BNO055 (calibration fine tuning is needed)

The python code is the reciver for the data the esp32 transmits via bluetooth and collects the data and maps it onto a 3d map head of the player (fine tuning is definetely needed for the data collection and mapping but the skeleton is there)
